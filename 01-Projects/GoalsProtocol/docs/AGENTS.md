### 🔴 Key Anti-Patterns (Embedded)

**From AGENT_LEARNINGS.md:**
1. **Don't** assume web services have public APIs
2. **Don't** ignore dependency warnings
3. **Don't** fire-and-forget async operations
4. **Don't** hardcode configuration

## 📝 Agent Memory System

### Short-Term (Session Context)
- Current task understanding
- Recent decisions and rationale
- Active file modifications

### Long-Term (Embedded Files)
- `AGENT_LEARNINGS.md` - Persistent mistake tracking
- `TRANSCENDENCE.md` - Capability elevation principles
- `ARCHITECTURE.md` - System design patterns
- `GOALS_PROTOCOL_SUMMARY.md` - Project overview

### Memory Update Protocol

**When a mistake occurs:**
1. Analyze root cause
2. Document in `AGENT_LEARNINGS.md`
3. Extract pattern to avoid
4. Propose better approach
5. Verify fix prevents recurrence

**When capability improves:**
1. Identify new pattern/technique
2. Add to `TRANSCENDENCE.md`
3. Update quality gates
4. Share across agent instances

### Pre-Action Checklist
```bash
# Before any significant action:
pnpm test              # Verify existing functionality
pnpm typecheck         # TypeScript validation
pnpm lint              # Code style check
pnpm arch:check        # Architecture rule validation
```

### Code Style (Embedded)
- **Format:** Single quotes, async preferred, semicolons required
- **Naming:** PascalCase types, camelCase functions, UPPER_SNAKE_CASE constants
